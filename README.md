# Document Builder Platform

**Document Builder** is a centralized platform for generating documents in a controlled and reproducible environment.

The goal of the project is to provide a **single, consistent document generation system** that allows employees to create documents easily while ensuring that all outputs follow the same structure, formatting, and build process.

---

## Core Principles

The platform is built around three fundamental requirements:

1. **Consistency**
   All documents should have a consistent structure, layout, and formatting.

2. **Accessibility**
   Every employee should be able to create documents without installing or maintaining a full LaTeX toolchain.

3. **Centralized generation**
   All documents are generated by a shared central builder to guarantee reproducibility and uniform output.

---

## Architecture Overview

The system is composed of three main layers.

```
Workstation
   │
   ▼
document-builder-client
   │
   ▼
document-builder-engine (API)
   │
   ▼
Queue / Redis
   │
   ▼
Builder Runner
   │
   ▼
LaTeX build environment
   │
   ▼
PDF artifact
```

Users prepare the document locally and send it to the builder.
The builder compiles the document and returns a generated PDF.

---

## Repositories

The platform is organized into several repositories.

### document-builder-engine

Central build engine responsible for:

* API service
* job lifecycle management
* build runner
* artifact storage
* queue management

### document-builder-client

Workstation client used to submit documents to the builder and download generated PDFs.

Provides:

* CLI client
* Windows and Linux launchers
* simple integration with local document projects

### document-builder-doc

Project documentation including:

* architecture description
* deployment instructions
* workstation setup
* troubleshooting guides

### document-builder-test-doc

Example document project used for integration testing and demonstrating the platform workflow.

---

## Typical Workflow

1. User prepares a document locally.
2. The workstation client submits the document to the builder.
3. The builder executes the document build process.
4. The resulting PDF artifact is returned to the workstation.

This ensures that every document is built in a **controlled and consistent environment**.
